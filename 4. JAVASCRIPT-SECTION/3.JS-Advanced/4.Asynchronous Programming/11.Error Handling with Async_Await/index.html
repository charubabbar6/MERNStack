Error Handling with Async/Await in JavaScript

Introduction

Error handling in asynchronous programming is crucial for building robust applications. While Promises in JavaScript offer the .catch() method for handling errors, the async/await syntax introduces more straightforward error handling using try, catch, and finally blocks.

Basic Error Handling

The simplest way to handle errors when using async/await is to wrap the await call within a try and catch block. 


async function fetchData() {
    try {
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Error fetching data:', error);
    }
}

The Finally Block

The finally block can be added after catch to execute code regardless of whether an error was thrown or not. 


async function fetchData() {
    try {
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Error fetching data:', error);
    } finally {
        console.log('Fetch attempt completed.');
    }
}

Handling Multiple Async Operations

When handling multiple asynchronous operations, each await expression can be enclosed in its own try/catch block for granular error control. 


async function performTasks() {
    try {
        const task1 = await doTask1();
        console.log(task1);
    } catch (error) {
        console.error('Task 1 failed:', error);
    }

    try {
        const task2 = await doTask2();
        console.log(task2);
    } catch (error) {
        console.error('Task 2 failed:', error);
    }
}

Conclusion

Using async/await with try, catch, and finally blocks provides an intuitive and clean approach to handling errors in asynchronous JavaScript code.
