Making HTTP Requests with the Fetch API

Introduction

The Fetch API provides a modern approach to make HTTP requests and handle responses. It returns Promises and is much cleaner and more powerful than the older XMLHttpRequest.

Basic GET Request

To perform a GET request, you can use the fetch() function: 


fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.log('Error:', error));

Using async/await

You can also use the Fetch API with async/await for better readability: 


async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.log('Error:', error);
  }
}

fetchData();

POST Request

To make a POST request, you need to pass an options object to the fetch() function: 


fetch('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ key: 'value' })
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.log('Error:', error));

Conclusion

Understanding how to use the Fetch API for making HTTP requests will help you manage client-server interactions in a more modern and cleaner way. It is more flexible and powerful than its predecessor, XMLHttpRequest, and is widely used in modern web development.
